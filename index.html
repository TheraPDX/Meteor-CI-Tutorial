Getting Started with Meteor Continuous Integration

These explanatory scripts, videos and slideshows, will get you started easily with Meteor package development, testing, documenting, code style linting and continuous integration

They are designed for small (10GB) Xubuntu 14.04 virtual machines. They have not been tested in any other environment since the idea is to provide clear proven instructions that can be adapted easily to other circumstances. Once you see how it works in one environment setting up for a different one should present few difficulties.

The goal is to help Meteor developers put together a complete continuous integration work environment with :

  • application
  • packages
  • version control
  • logging
  • documentation
  • unit tests
  • end-of-end tests
  • code quality assurance tests
  • deployment

Modus Operandi

Most likely you'll want to flip through the slides to get an overview, then watch the video, then spin up a fresh virtual machine and run through it all. It shouldn't cost you more than an hour or two. The slides change with the keyboard only, and they have hyperlinks to deeper details that work only poorly if the slides are also enabled for control by mouse/touchscreen.

Except where indicated, the bash scripts are 'idempotent' -- you can run them repeatedly without adverse consequences.

Prepare the Machine

Start off with a fresh Ubuntu 14.04 LTS virtual machine and install all the platform dependencies. These are global installs requiring 'sudo'. Subsequent stages should not require 'sudo'. The script shown can run to completion without intervention, or stop at intermediate steps to view explanations.

Unit Test the Package

This section follows the manual steps to perform in a project directory to begin testing with a version controlled Meteor package.

You will prepare a Meteor project with an embedded package ready for testing with TinyTest using some of the application development support tools installed in the previous section. It includes the minimal basic set up tasks of a Meteor developer's work station with linting, unit testing and version control.

First steps

To begin, you will need to obtain the tutorial. Open up a terminal window, then copy'n paste these commands into it:

export URI="https://github.com/martinhbramwell"
export TUT="Meteor-CI-Tutorial"
export FILE="archive/master.zip"

echo "Downloading : \n${URI}/${TUT}/${FILE}"
echo "         as : ${TUT}.zip"

wget -O ${TUT}.zip ${URI}/${TUT}/${FILE}
unzip ${TUT}.zip
mv ${TUT}-master ${TUT}
cd ${TUT}

# Ready to run the first script :
#             sudo ./Part01_PrepareTheMachine.sh

The result should look like this.

Then you can run the first script with:

sudo ./Part01_PrepareTheMachine.sh

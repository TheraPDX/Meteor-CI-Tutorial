Getting Started with Meteor Continuous Integration

These scripts, and explanatory slideshows, prepare the way for you to get started easily with Meteor package development, testing, documenting, code style linting and continuous integration

They are designed for small (10GB) Xubuntu 14.04 virtual machines. They have not been tested in any other environment since the idea is to provide clear proven instructions that can be adapted easily to other circumstances.

Except where indicated, the scripts are 'idempotent' -- you can run them repeatedly without adverse consequences.

The goal is to help Meteor developers put together a complete continuous integration work environment with :

  • application
  • packages
  • version control
  • logging
  • documentation
  • code quality assurance tests
  • deployment

Prepare the Machine

Start off with a fresh Ubuntu 14.04 LTS virtual machine and install all the platform dependencies. These are global installs requiring 'sudo'. Subsequent stages should not require 'sudo'. The script shown can run to completion without intervention, or stop at intermediate steps to view explanations.

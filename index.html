Getting Started with Meteor Continuous Integration

These scripts, video and explanatory slideshows, will get you started easily with Meteor package development, testing, documenting, code style linting and continuous integration

They are designed for small (10GB) Xubuntu 14.04 virtual machines. They have not been tested in any other environment since the idea is to provide clear proven instructions that can be adapted easily to other circumstances.

The goal is to help Meteor developers put together a complete continuous integration work environment with :

  • application
  • packages
  • version control
  • logging
  • documentation
  • code quality assurance tests
  • deployment

Modus Operandi

Most likely you'll want to flip through the slides to get an overview, then watch the video, then spin up a fresh virtual machine and run through it all. It shouldn't cost you more than an hour. The slides change with the keyboard only, and they have hyperlinks to deeper details that work only poorly if the slides are also enabled for control by mouse/touchscreen.

Except where indicated, the bash scripts are 'idempotent' -- you can run them repeatedly without adverse consequences.

Prepare the Machine

Start off with a fresh Ubuntu 14.04 LTS virtual machine and install all the platform dependencies. These are global installs requiring 'sudo'. Subsequent stages should not require 'sudo'. The script shown can run to completion without intervention, or stop at intermediate steps to view explanations.
